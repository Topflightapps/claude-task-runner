## Codebase Patterns
- Config uses Zod schema in `src/config.ts` with `z.object().transform()` pattern
- All config values come from `process.env` (strings) - use `.coerce` or `.transform()` for non-string types
- Optional config fields use `.optional()`, required ones use `.min(1)`
- Config is a lazy singleton via `loadConfig()` / `getConfig()`
- DB uses `better-sqlite3` with `getDb()` accessor in `src/db.ts`
- Typecheck: `npx tsc --noEmit`
- ESLint: Use `String()` for numbers in template literals (e.g. `${String(status)}`)
- ESLint: Don't use optional chain (`?.`) on values TS considers non-nullish — use `as` type assertions with care
- Librarian modules live in `src/librarian/`
- ESLint `perfectionist/sort-modules` requires exported functions alphabetically sorted
- Don't annotate types on params with defaults (e.g. `limit = 10` not `limit: number = 10`)
- ESLint `perfectionist/sort-union-types`: `null` comes before other types (e.g. `null | string`)
- ESLint `perfectionist/sort-modules`: private functions must also be alphabetically sorted within their group
- For simple Claude calls, use `promisify(execFile)` with timeout — strip `CLAUDECODE`/`CLAUDE_CODE_SESSION` env vars
- Use `createChildLogger("module-name")` from `src/logger.js` — never use `console.log/error/warn` (ESLint `no-console`)
- Quality hook validates each edit — add imports and their usage in the same edit to avoid unused-import errors
- `ClickUpTask.description` is `string` (non-nullable) — don't use `?? ""` on it
- `handleAdminApi` is sync (not async) — use static imports, not dynamic `await import()`
- Admin API paginated endpoints return `{ rows, total }` using COUNT query + LIMIT/OFFSET
